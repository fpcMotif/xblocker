## 2024-05-18 - Contextual ARIA Labels in Lists & Status Roles
**Learning:** Screen readers often struggle with repetitive action buttons in lists (like a row of "Remove" buttons). Without context, a screen reader user just hears "Remove, Remove, Remove". Similarly, input fields inside modals that are described by paragraphs often lose that context unless explicitly tied with `aria-describedby`. Finally, dynamically added elements like Toasts need `role="status"` and `aria-live="polite"` or they are never read aloud.
**Action:** When adding repeated buttons in lists (like in the whitelist manager), always interpolate the target item into the `aria-label` (e.g., `aria-label="Remove @username from whitelist"`). Use `aria-describedby` to associate helper text with inputs. Use `role="status"` and `aria-live="polite"` for non-disruptive toast notifications.
